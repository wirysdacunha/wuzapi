API REST para WhatsApp

WUZAPI é uma implementação da biblioteca whatsmeow como um serviço de API RESTful simples, com suporte a múltiplos dispositivos e sessões concorrentes.

Principais Recursos

O WUZAPI oferece uma API completa para interagir com o WhatsApp de forma eficiente e sem usar recursos pesados como Puppeteer ou emuladores Android.

Alta Performance

Comunicação direta com os servidores do WhatsApp via websocket, resultando em menor uso de memória e CPU.

Múltiplas Sessões

Suporte a múltiplos dispositivos e usuários simultaneamente na mesma instância.

Mensagens Ricas

Envie texto, imagens, áudio, documentos, vídeos, adesivos, localização e contatos.

Webhooks

Configure webhooks para receber notificações em tempo real de eventos e mensagens.

Verificação de Usuários

Verifique se números de telefone têm WhatsApp e obtenha informações de perfil.

Autenticação Simples

Sistema de tokens para autenticação fácil e segura de usuários.

  • Verificar status de conexão
  • Obter QR Code para escaneamento

Chat

  • Marcar mensagens como lidas

Grupos

Gerencie grupos do WhatsApp.

  • Listar grupos inscritos
  • Obter informações de grupo
  • Obter link de convite
  • Alterar nome e foto do grupo

Webhooks

Configure notificações para eventos.

  • Definir URL do webhook
  • Subscrever eventos específicos
  • Atualizar configuração
  • Remover webhook

Como Utilizar

Instruções para começar a usar o WUZAPI rapidamente.

Pré-requisitos

Para executar o WUZAPI, você precisa de:

  • Go (Linguagem de Programação Go)
  • Docker (opcional, para containerização)

Compilação

Compile o projeto com o seguinte comando:

go build .

Execução

Por padrão, o serviço iniciará na porta 8080. Você pode alterar o comportamento com os seguintes parâmetros:

  • -address: define o endereço IP para vincular o servidor (padrão 0.0.0.0)
  • -port: define o número da porta (padrão 8080)
  • -logtype: formato para logs, console (padrão) ou json
  • -wadebug: habilita debug do whatsmeow, níveis INFO ou DEBUG são suportados
  • -sslcertificate: Arquivo de Certificado SSL
  • -sslprivatekey: Arquivo de Chave Privada SSL

Exemplo:

./wuzapi -logtype json

Criando Usuários

Para abrir sessões, primeiro você precisa criar um usuário e definir um token de autenticação. Faça isso atualizando o banco de dados Postgres users.db:

psql -h "$DB_HOST" -p "$DB_PORT" -U "$DB_USER" -d "$DB_NAME" -c "INSERT INTO users (name, token) VALUES ('John','1234ABCD');"

Uma vez criados os usuários, você pode se comunicar com a API passando o cabeçalho Token como meio simples de autenticação. Você pode ter vários usuários (números diferentes) no mesmo servidor.

Recursos Web

O daemon também serve alguns arquivos web estáticos, úteis para desenvolvimento/teste que você pode carregar com seu navegador:

  • Uma referência swagger da API em /api
  • Uma página web de exemplo para conectar e escanear códigos QR em /login (onde você precisará passar ?token=1234ABCD)
  • Um painel de gerenciamento em /dashboard
